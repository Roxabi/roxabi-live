## Problem

Follow-up from PR #55 code review. The `/api/*` endpoints and the GitHub webhook are reachable publicly through the cloudflared tunnel at `dashboard.roxabi.dev` with no defense-in-depth. `corpus.db` contains issues from private repos, yet any unauthenticated client can query them. The webhook reads the request body unbounded before HMAC verification (DoS amplifier), path parameters on `/api/issues/{key:path}` are passed to the DB layer without validation, and the reconciler retries indefinitely on GitHub auth failures instead of stopping and alerting.

Four concrete findings (SEC-1 through SEC-4) from the review need to land before the tunnel sees broader use.

## Who

- **Primary:** Mickael — operator of `dashboard.roxabi.dev`; wants private-repo data not to leak and the webhook path not to be a DoS vector.
- **Secondary:** future readers of the API (any additional dashboard tabs); the reconciler on-call path (silent auth failures today = silent data staleness).

## Constraints

- Cloudflare Access must be configured via the Cloudflare dashboard/API; tunnel hostname stays `dashboard.roxabi.dev`.
- Webhook HMAC verify must stay first-check after size gate (no early DB or auth lookups before signature is valid).
- Body-size cap ~25 MB (GitHub's webhook payload ceiling) — must return 413 before reading full body.
- Path validation regex: `^[A-Za-z0-9._-]+/[A-Za-z0-9._-]+#[0-9]+$` — 400 on mismatch.
- Reconciler must distinguish transient (5xx, network) from auth (401/403) errors; repeated auth failures stop the scheduler and emit CRITICAL.
- Stack: FastAPI + aiosqlite. Prefer framework-native middleware over custom shims.

## Out of Scope

- Full auth system for the API (Cloudflare Access is the gate — no per-user app auth).
- Rate limiting beyond body-size cap.
- Schema changes to `corpus.db`.
- Rewriting the reconciler; only its error-handling branch.
- Frontend/dep-graph changes.

## Complexity

**Tier: F-lite** — clear scope, single backend domain + one infra config step, 4 concrete findings already enumerated with acceptance criteria. No architectural unknowns.

Signals:

- One infra step (Cloudflare Access policy) outside the codebase.
